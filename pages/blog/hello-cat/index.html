Relational Database

A relational database is just a collection of flat files (tables) that have relationships with each other. There are two basic building blocks to a relational database: tables and fields.Tables have fields (aka columns). These fields have relationship to fields in other tables. How do we show these relationships.

A nice simple way of doing this is to take each table and make a rectangle out of it, listing all the fields we know about. For example if we have two tables called Student and Class. We use the datamodelr package to visualizae our models.

Code 
library(datamodelr)
## dm stands for data model
dm = dm_from_data_frames(Student, Class)
graph = dm_create_graph(dm, rankdir = "BT", col_attr = c("column", "type"))
dm_render_graph(graph)

In this case, each student takes several classes. How do we represent these relationship? In come primary and foreign keys.

  • Student.StudentID is a primary key because every student will have a different StudentID
  • Class.ClassID is a primary key because every class will have a different ClassID
  • Student.ClassID is a foreign key because it is a field used to establish a link between data in two datables Student and Class. We can use ClassID to show what classes a student is taking and use it to link to information about that class.

This relationship can be shown as


  +-------------+                 +-------------+
  |  Student    |                 |   Class     |
  +-------------+                 +-------------+
  | StudentID   |                 | ClassID     |
  | ClassID     |                 | Teacher     |
  | DOB         |                 | Location    |
  +-------------+                 +-------------+

## Relational Database

A relational database is just a collection of flat files (tables) that have relationships with each other. How do we show these relationships.
